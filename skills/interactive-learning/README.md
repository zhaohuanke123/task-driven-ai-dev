# Interactive Learning

交互式学习系统，基于Bloom的"2 Sigma问题"理论和掌握学习原则。

## 概述

这个skill实现了一个结构化的交互式学习流程，帮助AI作为一对一辅导员：

- 根据用户选择的主题创建学习路径
- 生成结构化的学习文件
- 通过检查站问题评估掌握程度
- 根据反馈自适应调整学习进度
- 持续追踪学习进度和误区

## 理论基础

基于Benjamin Bloom的"2 Sigma问题"研究：一对一辅导的学生比普通课堂学生表现高出2个标准差（超越96%的课堂学生）。

**核心原则：掌握学习（Mastery Learning）**——学生真正理解内容后，才推进到下一阶段。

## 安装

将此skill复制到你的Claude Code skills目录：

```bash
# 项目级使用
cp -r skills/interactive-learning ~/.claude/skills/

# 或者保持在项目中
# 当你在该项目工作时，skill会自动可用
```

## 快速开始

1. **启动学习**：

```
我想学习Docker容器技术
```

或者

```
教我Python基础
```

2. **回答检查站问题**：

学习文件末尾有检查站问题，直接回答即可。系统会根据你的回答判断掌握程度。

3. **自适应推进**：

- 完全掌握 → 推进下一课
- 基本掌握 → 简短纠正后推进
- 部分理解 → 补充材料巩固
- 尚未理解 → 从基础重新解释

## 学习文件结构

每个学习文件包含：

| 部分 | 说明 |
|------|------|
| 标题 | 课程主题和编号 |
| 学习目标 | 本节结束后能做到什么 |
| 内容讲解 | 核心概念，使用类比和例子 |
| 小结 | 3-5条关键点 |
| 检查站 | 2-4个问题，验证掌握程度 |

## 教学风格

1. **费曼技巧** - 用简单语言解释复杂概念
2. **苏格拉底式提问** - 用问题引导思考
3. **脚手架原则** - 从已知推向未知
4. **鼓励犯错** - 错误是诊断理解深度的工具

## 示例对话

**用户**：我想学习Git版本控制

**Claude**：
1. 创建 `Git版本控制\` 文件夹
2. 生成 `进度.md` 和 `01_Git基础概念.md`
3. 展示学习内容，结尾有检查站问题

**用户**：回答检查站问题...

**Claude**：
- 判断掌握程度
- 给出反馈
- 生成下一课或补充材料
- 更新进度
